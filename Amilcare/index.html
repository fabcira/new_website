Caveats Chamges since V3.2

  1. The most important aspect is that the error thresholds is now defined as correctMatches/matches. Until now it was defined as errors/matched. This means that what in version 2.x was expected to be as error threshold 0.1 in V3.x must be declared as 0.9
  2. From V3.x onward there were some substantial changes wrt the previous versions not reported in the interface for the moment!!!!.

Known Errors

If you receive a Gate error during initialization of the preprocessor with a java error trace (generally mentioning a V8 problem), try to download the following file, unzip it and put it in your JAVA_HOME/lib. If that file is already existent, replace it.

  • tools.jar

Be sure that the file tools.jar is in your classpath, otherwise it will not work!!!!
